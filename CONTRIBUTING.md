# Contributing to MeetSpectatorLive

Thank you for considering contributing to MeetSpectatorLive! We appreciate your help in making our project better. This guide provides important information about the contribution process.

## Guidelines for Contributors
- Be respectful and considerate of others in the community.
- Follow the project's code of conduct.

## Code Style
- We follow [Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript).
- Make sure to run `eslint` to check for code style issues.

## Testing Requirements
- All contributions must include appropriate tests.
- To run the tests, use the command `npm test`.
- Maintain a 100% code coverage unless otherwise discussed with a project maintainer.

## Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Make your changes and ensure they pass all tests.
3. Commit your changes with a clear message describing your modifications.
4. Push your branch to your forked repository.
5. Open a pull request against the `main` branch of this repository.

## Development Workflow
- Regularly pull changes from `main` to keep your branch up to date.
- Use descriptive commit messages to make it easier for reviewers.
- Engage with reviewers during the pull request review process for efficient collaboration.

Thank you for contributing!
